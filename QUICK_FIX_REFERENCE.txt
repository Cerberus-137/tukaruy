================================================================================
                          ROOT CAUSE - THE FIX
================================================================================

THE CRITICAL FIX (in .htaccess):

BEFORE (Wrong - causes API failures):
  RewriteCond %{REQUEST_URI} !^/api/
  RewriteCond %{REQUEST_URI} !^/admin/api/
  RewriteRule ^(.*)$ $1.php [L]

AFTER (Correct - stops API rewriting):
  RewriteRule ^(api|admin/api)($|/) - [L]
  RewriteCond %{REQUEST_FILENAME} !-d
  RewriteCond %{REQUEST_FILENAME}\.php -f
  RewriteRule ^(.*)$ $1.php [L]

Why it works:
  • RewriteRule matches BEFORE conditions take effect
  • Using - [L] stops ALL processing for /api/ paths
  • Subdirectory .htaccess files then handle clean URLs properly
  • Result: API files execute with JSON, not HTML errors
